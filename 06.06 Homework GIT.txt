1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout (file-name)

2.Как вернуть файл из индекса в рабочий каталог?
git restore --staged (file name) 

3.Как отменить ошибочный commit ?
git reset HEAD ~ (number)

4.Как создать новую ветку и переключится в нее?
git checkout -b (branch name)

5) В чем отличение между fast-forward merge and non fast-forfard merge?
Fast-forward merge – быстрое перемещение, когда мы хотим слить ветку на которой были произведены изменения с веткой без изменений.
Non fast-forward - когда мы собираемся сливать две ветки, и на обоих происходили изменения (на каждой ветке разные изменения)

6) Когда может возникнуть конфликт слияния веток, как его разрешить?
Когда происходит конфликт слияния веток, гит предлагает нам открыть файл, в котором на разных ветках были произведены разные изменения. В этом файле будут содержаться все изменения из двух веток. Гит предложит выбрать, какие изменения оставить, а какие удалить, или же ничего не удалять. Для этого удаляем ненужные символы(сгенерированные гитом). Корректируем файл и выходим.
git status
git merge – abort    (если хотим отменить слияние)
git commit -a    или    git add   (чтобы закончить слияние)

7.Можно ли удалить ветку? Как? 
git branch -d (branch name)
